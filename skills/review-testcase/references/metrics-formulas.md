# 量化指标计算公式

本文档定义 review-testcase 技能的7项量化指标的计算公式、合格线、数据来源，以及缺少参考数据时的处理方式。

---

## 指标总览

| 序号 | 指标 | 合格线 | 需要额外数据 |
|------|------|--------|-------------|
| 1 | 需求覆盖率 | ≥ 95% | ✅ PRD文档 |
| 2 | 反向用例占比 | ≥ 30% | ❌ |
| 3 | 边界用例数量 | 每功能点 ≥ 2个 | ❌ |
| 4 | 重复用例率 | ≤ 10% | ❌ |
| 5 | 错误用例率 | ≤ 5% | ❌ |
| 6 | 高危场景覆盖率 | 100% | ✅ 高危场景清单 |
| 7 | 历史缺陷覆盖率 | ≥ 80% | ✅ 历史Bug清单 |

---

## 指标1：需求覆盖率

### 计算公式

```
需求覆盖率 = 已覆盖功能点数 / PRD总功能点数 × 100%
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| PRD总功能点数 | 从PRD文档中提取 | 必需 |
| 已覆盖功能点数 | AI对比用例与PRD功能点 | 必需 |

### 功能点识别规则

1. **PRD功能点提取**：
   - PRD中每个用户故事/功能需求视为1个功能点
   - 每个功能需求下的子规则视为子功能点
   - 联动规则单独计为1个功能点
   - 计数公式：总功能点 = 主功能点数 + 子功能点数 + 联动规则数

2. **用例覆盖判定**：
   - 每个功能点至少有1条用例覆盖即算"已覆盖"
   - "覆盖"定义：用例的测试目标与功能点对应，且步骤中涉及该功能点的核心操作
   - 仅在标题中提及但步骤中未涉及的不算覆盖

### 缺少PRD时的处理

当用户未提供PRD文档时：
- 标记为"⚠️ 缺少PRD文档，无法计算需求覆盖率"
- 建议用户提供PRD文档以获取完整评审

---

## 指标2：反向用例占比

### 计算公式

```
反向用例占比 = 反向/异常用例数 / 总用例数 × 100%
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| 总用例数 | 从用例文件中统计 | 用例文件 |
| 反向/异常用例数 | AI语义识别 | 用例文件 |

### 反向/异常用例识别规则

**自动识别（关键词匹配）**：

| 分类 | 识别关键词 |
|------|-----------|
| 异常输入 | 错误、无效、非法、异常、不合法、不符合、格式错误、超长、超短、空值、空格、特殊字符 |
| 边界值 | 边界、最大值、最小值、上限、下限、临界、刚好、超出 |
| 业务异常 | 失败、拒绝、不允许、不可、禁止、超时、余额不足、权限不足、已存在、不存在 |
| 错误处理 | 提示、报错、错误信息、异常处理、容错、降级、重试 |
| 状态异常 | 重复、过期、已取消、已删除、已关闭、互斥、冲突 |

**AI语义识别（当关键词匹配不足时）**：

1. 分析用例预期结果：预期结果描述"提示xxx错误""显示xxx信息""不允许xxx操作"的为反向用例
2. 分析用例标题：标题含"异常""失败""错误""非法""无效"等语义的为反向用例
3. 分析测试步骤：步骤中包含触发异常的操作（如输入超长字符串、未登录访问）的为反向用例

### 合格判定

- ≥ 30%：✅ 达标
- 20-29%：⚠️ 偏低，建议补充反向用例
- < 20%：❌ 不达标，反向用例严重不足

## 指标3：边界用例数量

### 计算公式

```
边界用例数量 = 含明确边界值测试的用例总数
每功能点边界用例数 = 边界用例总数 / 功能点总数
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| 边界用例总数 | AI语义识别 | 用例文件 |
| 功能点总数 | 从用例中识别（或PRD） | 用例文件/PRD |

### 边界用例识别规则

**识别信号词**：
```
边界、最大、最小、上限、下限、临界、刚好、超出、不足、
0、-1、空、null、undefined、满、溢出、超限、极端
```

**具体判断标准**：

1. 用例测试了输入值的最小值/最大值 → 边界用例
2. 用例测试了刚好满足/刚好不满足条件的值 → 边界用例
3. 用例测试了空值/null/0/负数 → 边界用例
4. 用例测试了超出范围的值（如超长字符串、超大数值） → 边界用例
5. 用例测试了并发数的上限 → 边界用例

**功能点统计**：

- 如果有PRD文档：按PRD功能点统计
- 如果无PRD文档：按用例中涉及的独立功能模块统计（如"登录""支付""订单"等）

### 合格判定

- 每功能点 ≥ 2个：✅ 达标
- 每功能点 1个：⚠️ 偏低
- 每功能点 0个：❌ 不达标，缺少边界用例

## 指标4：重复用例率

### 计算公式

```
重复用例率 = 语义重复用例数 / 总用例数 × 100%
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| 总用例数 | 从用例文件中统计 | 用例文件 |
| 语义重复用例数 | AI语义比对 | 用例文件 |

### 语义重复判定规则

**判定流程**：

1. **标题比对**：两条用例的标题核心语义一致（去除编号和修饰词后）
2. **步骤比对**：测试步骤的核心操作序列一致
3. **数据比对**：测试数据相同（或仅编号不同）
4. **预期结果比对**：预期结果描述的验证点一致

**完全重复**（归为第三类-错误无效）：
- 标题+步骤+数据+预期结果核心语义全部一致
- 仅编号/标题措辞不同

**部分重复**（不归为错误，但计入重复率）：
- 步骤和数据相似，但前置条件不同 → 不算重复
- 步骤相似，但测试数据不同 → 不算重复
- 同一功能点的正向和反向用例 → 不算重复

**重复组识别**：
- 将所有重复用例标记为一个"重复组"
- 每组保留1条代表，其余计为重复
- 重复用例数 = 所有重复组中除代表外的用例总数

### 合格判定

- ≤ 10%：✅ 达标
- 11-20%：⚠️ 偏高，建议去重
- > 20%：❌ 不达标，重复用例过多

---

## 指标5：错误用例率

### 计算公式

```
错误用例率 = 第三类（错误无效）用例数 / 总用例数 × 100%
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| 总用例数 | 从用例文件中统计 | 用例文件 |
| 第三类用例数 | Phase 2 分类结果 | AI评审 |

### 合格判定

- ≤ 5%：✅ 达标
- 6-10%：⚠️ 偏高，AI用例生成质量需提升
- > 10%：❌ 不达标，建议重新生成

---

## 指标6：高危场景覆盖率

### 计算公式

```
高危场景覆盖率 = 已覆盖高危场景数 / 高危场景总数 × 100%
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| 高危场景总数 | 用户提供的清单 | 必需 |
| 已覆盖高危场景数 | AI比对用例与清单 | 必需 |

### 高危场景默认清单

当用户未提供高危场景清单时，使用以下默认识别规则：

**自动识别的高危场景类型**：

| 高危类型 | 识别关键词 | 必须覆盖 |
|---------|-----------|---------|
| 支付相关 | 支付、扣款、退款、充值、提现 | ✅ |
| 订单流转 | 下单、取消、退货、换货 | ✅ |
| 权限控制 | 越权、权限、角色、管理员 | ✅ |
| 数据安全 | 加密、脱敏、隐私、敏感信息 | ✅ |
| 并发竞态 | 并发、锁、互斥、超卖 | ✅ |
| 金额计算 | 优惠、折扣、满减、运费 | ✅ |

### 缺少高危场景清单时的处理

- 按默认识别规则自动识别高危场景
- 标记为"⚠️ 未提供高危场景清单，使用默认识别规则"
- 建议用户提供完整的场景清单以获取更准确的覆盖率

---

## 指标7：历史缺陷覆盖率

### 计算公式

```
历史缺陷覆盖率 = AI用例能覆盖的历史Bug数 / 近3版本Bug总数 × 100%
```

### 数据来源

| 数据项 | 来源 | 必要性 |
|--------|------|--------|
| 近3版本Bug总数 | 用户提供的Bug清单 | 必需 |
| AI用例能覆盖的Bug数 | AI比对用例与Bug场景 | 必需 |

### Bug覆盖判定规则

**"覆盖"定义**：存在至少1条用例，其测试场景能复现或验证该Bug的触发条件和表现

**判定流程**：

1. 提取每条Bug的核心要素：触发条件、错误表现、影响范围
2. 在用例中搜索是否存在场景匹配：
   - 触发条件匹配：用例步骤能复现Bug的触发条件
   - 表现验证：用例的预期结果包含对Bug表现的验证
3. 任一匹配即算"已覆盖"

### 缺少历史Bug清单时的处理

- 标记为"⚠️ 缺少历史Bug清单，无法计算历史缺陷覆盖率"
- 建议用户提供近3个版本的线上Bug清单

## 无额外数据时的最少可用指标

当用户仅提供用例文件，未提供PRD、高危场景清单、历史Bug清单时，仍可计算的指标：

| 指标 | 是否可计算 | 备注 |
|------|-----------|------|
| 需求覆盖率 | ❌ | 需PRD文档 |
| 反向用例占比 | ✅ | AI语义识别 |
| 边界用例数量 | ✅ | AI语义识别（功能点从用例推断） |
| 重复用例率 | ✅ | AI语义比对 |
| 错误用例率 | ✅ | 基于分类结果 |
| 高危场景覆盖率 | ⚠️ 部分 | 使用默认识别规则 |
| 历史缺陷覆盖率 | ❌ | 需历史Bug清单 |

**最少可用指标数：4项（完全可计算）+ 1项（部分可计算）**
