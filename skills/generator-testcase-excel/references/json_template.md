# JSON 数据模板参考

本文档定义了测试用例 JSON 数据的结构规范，用于 Excel 生成脚本的输入。

---

## 完整结构定义

```json
{
    "title": "项目名称 - 测试用例",
    "test_cases": [
        {
            "id": "TC-001",
            "title": "用例标题",
            "precondition": "前置条件描述",
            "steps": "1. 步骤一\n2. 步骤二\n3. 步骤三",
            "test_data": "具体测试数据描述",
            "expected_result": "预期结果描述",
            "priority": "P0",
            "requirement_id": "US-001"
        }
    ]
}
```

## 字段说明

| 字段 | 必填 | 说明 | 示例 |
|------|------|------|------|
| id | 是 | 用例编号，全局连续 | TC-001 |
| title | 是 | 用例标题，简洁明确 | 手机号注册-输入合法手机号和验证码注册成功 |
| precondition | 是 | 执行前需满足的条件 | 用户未注册过账号 |
| steps | 是 | 测试步骤，编号列表 | 1. 打开注册页面\n2. 输入手机号 13812345678\n3. 点击获取验证码 |
| test_data | 是 | 具体测试数据 | 手机号：13812345678，验证码：123456，密码：Abc12345 |
| expected_result | 是 | 预期结果 | 1. 注册成功提示\n2. 自动跳转首页\n3. 显示用户昵称 |
| priority | 是 | 优先级 P0/P1/P2 | P0 |
| requirement_id | 是 | 关联需求编号 | US-001 |

## 编号规则

- 测试用例编号全局连续：TC-001, TC-002, TC-003, ...
- 优先级字段仅允许：P0, P1, P2
