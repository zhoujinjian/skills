### 1.2 关键词匹配规则

1. **精确匹配**：用户描述中包含关键词列表中的任一词
2. **模糊匹配**：用户描述中包含关键词的子串（如"身份证"匹配"身份证号"）
3. **优先级**：精确匹配 > 模糊匹配；更具体的关键词优先（如"身份证号" > "身份证"）
4. **"等"字处理**：描述末尾有"等"表示还有更多字段，使用核心字段列表补全

### 1.3 默认字段集

当用户描述中使用"等"字或未明确指定所有字段时，使用以下默认字段集：

| 场景 | 默认字段集 |
|------|-----------|
| "用户数据" | 姓名、手机号、邮箱、用户名 |
| "注册数据" | 用户名、密码、邮箱、手机号 |
| "收货数据"/"地址数据" | 姓名、手机号、地址、省份、城市、邮编 |
| "公司数据"/"企业数据" | 公司名、地址、城市、电话、邮箱 |
| "测试数据"（无具体字段） | 姓名、手机号、邮箱、地址 |
| "性能测试数据" | 用户名、密码、邮箱、手机号 |

### 2.3 自定义生成器

以下字段类型使用自定义逻辑生成（Faker 没有对应的中文 Provider）：

#### 密码

```python
def generate_password():
    """生成满足常见密码规则的密码：8-20位，含大小写字母、数字、特殊字符"""
    import random, string
    length = random.randint(8, 16)
    chars = [
        random.choice(string.ascii_uppercase),
        random.choice(string.ascii_lowercase),
        random.choice(string.digits),
        random.choice("!@#$%^&*"),
    ]
    remaining = ''.join(random.choices(
        string.ascii_letters + string.digits + "!@#$%^&*", k=length - 4
    ))
    password = list(chars[0] + chars[1] + chars[2] + chars[3] + remaining)
    random.shuffle(password)
    return ''.join(password)
```

#### 车牌号

```python
def generate_license_plate():
    """生成中国车牌号：省份简称 + 字母 + 5位字母数字"""
    provinces = "京沪津渝冀豫云辽黑湘皖鲁新苏浙赣鄂桂甘晋蒙陕吉闽贵粤川青藏琼宁"
    import random, string
    province = random.choice(provinces)
    letter = random.choice(string.ascii_uppercase)
    suffix = ''.join(random.choices(string.ascii_uppercase + string.digits, k=5))
    return f"{province}{letter}·{suffix}"
```

## 3. 合法数据生成规则

### 3.1 各字段类型的合法数据特征

| 字段类型 | 合法特征 | 示例 |
|---------|---------|------|
| 身份证 | 18位、地区码+出生日期+顺序码+校验位、校验位正确 | `110101199001011234` |
| 手机号 | 11位、1开头、第二位3-9 | `13800001234` |
| 用户名 | 3-20位字母数字下划线 | `testuser01` |
| 姓名 | 2-4个中文字 | `张伟` |
| 邮箱 | 标准格式 `user@domain.tld` | `test@example.com` |
| 地址 | 省+市+区+详细地址 | `北京市朝阳区建国路1号` |
| 公司名 | 地区+行业+公司类型 | `北京创新科技有限公司` |
| 密码 | 8-20位，含大小写+数字+特殊字符 | `Test@1234` |
| URL | `http://` 或 `https://` 开头 | `https://example.com` |
| IP | `0-255.0-255.0-255.0-255` | `192.168.1.1` |
| 银行卡号 | 16位数字、Luhn 校验通过 | `4111111111111111` |
| 车牌号 | 省份简称+字母·5位字母数字 | `京A·12345` |

### 3.2 中国身份证校验位算法

```
校验位权重：[7, 9, 10, 5, 8, 4, 2, 1, 6, 3, 7, 9, 10, 5, 8, 4, 2]
校验码对应：['1', '0', 'X', '9', '8', '7', '6', '5', '4', '3', '2']

步骤：
1. 前17位分别乘以对应权重
2. 求和后对11取余
3. 余数对应校验码表中的字符即为校验位
```

### 4.2 非法数据生成策略

1. **格式变体**：改变合法数据的格式特征（增删改字符）
2. **边界变体**：长度边界值（刚好短1位、刚好长1位）
3. **类型变体**：替换为其他类型的数据（如手机号替换为邮箱格式）
4. **安全变体**：注入攻击载荷（SQL注入、XSS 等）
5. **空值变体**：空字符串、纯空格、null

### 4.3 非法数据数量分配

对于"生成 N 个非法数据"的请求，按以下比例分配：

| 变体类型 | 占比 | 数量（N=5时） |
|---------|------|-------------|
| 格式变体 | 40% | 2 |
| 边界变体 | 20% | 1 |
| 类型变体 | 10% | 0-1 |
| 安全变体 | 20% | 1 |
| 空值变体 | 10% | 0-1 |

## 5. 中文场景特殊处理

### 5.1 中国手机号号段规则

合法号段（2026年）：

| 运营商 | 号段 |
|--------|------|
| 中国移动 | 134-139、147、150-152、157-159、172、178、182-184、187-188、195、197、198 |
| 中国联通 | 130-132、145、155-156、166、171、175-176、185-186、196 |
| 中国电信 | 133、149、153、173-174、177、180-181、189、190、191、193、199 |

**生成规则**：从上述号段中随机选取前3位，后8位随机生成。

### 5.2 中国身份证号码规则

| 位置 | 含义 | 规则 |
|------|------|------|
| 1-6位 | 地址码 | 必须是有效的行政区划代码 |
| 7-14位 | 出生日期 | yyyyMMdd，日期必须合法 |
| 15-17位 | 顺序码 | 奇数分配给男性，偶数分配给女性 |
| 第18位 | 校验位 | 按校验位算法计算，可为X |

**常用地区码**：

| 地区码 | 地区 |
|--------|------|
| 110101 | 北京市东城区 |
| 110102 | 北京市西城区 |
| 310115 | 上海市浦东新区 |
| 440305 | 深圳市南山区 |
| 440106 | 广州市天河区 |
| 330102 | 杭州市上城区 |
| 510107 | 成都市武侯区 |

### 5.3 中文地址生成规则

生成的中文地址应遵循以下结构：

```
{省份}{城市}{区县}{街道/路}{门牌号}
```

示例：
- `北京市朝阳区建国路1号`
- `上海市浦东新区陆家嘴环路1000号`
- `广东省深圳市南山区科技园路8号`

### 5.4 公司名称生成规则

中文公司名通常遵循以下结构：

```
{地区}{字号}{行业}{组织形式}
```

- 地区：省/市名
- 字号：2-4个汉字
- 行业：科技、贸易、咨询、网络、信息、教育等
- 组织形式：有限公司、股份有限公司、集团有限公司等

示例：
- `北京创新科技有限公司`
- `上海瑞达贸易有限公司`
- `深圳远景网络技术有限公司`

## 6. CSV 输出规范

### 6.1 文件编码

- 编码：UTF-8 with BOM（兼容 Excel 直接打开不乱码）
- 换行符：LF（\n）
- 分隔符：逗号（,）

### 6.2 文件命名

```
nl_test_data_{字段类型}_{YYYYMMDD}_{HHMMSS}.csv
```

示例：
- `nl_test_data_ssn_20260526_113000.csv`
- `nl_test_data_phone_email_20260526_113000.csv`
- `nl_test_data_用户数据_20260526_113000.csv`

### 6.3 表头规则

- 首行为字段名（中文或英文，根据字段类型自动确定）
- 多字段时按用户描述的顺序排列
- 如果用户未指定字段顺序，按默认字段集顺序

### 6.4 数据行规则

- 每行一条数据
- 数值类型不加引号
- 字符串类型中的逗号用双引号包裹
- 空值用空字符串表示（非 null）

### 6.5 示例输出

#### 单字段输出

```csv
身份证号
110101199001011234
110101199002022345
440305199103033456
```

#### 多字段输出

```csv
姓名,手机号,邮箱,地址
张伟,13800001234,zhangwei@example.com,北京市朝阳区建国路1号
李娜,13800001235,lina@example.com,上海市浦东新区陆家嘴路2号
王磊,13800001236,wanglei@example.com,广东省深圳市南山区科技园路8号
```

#### 含合法/非法标记的输出

```csv
手机号,合法性,说明
13800001234,合法,标准手机号
15912345678,合法,移动号段
23800001234,非法,非1开头
1380000123,非法,10位少1位
1380000abcd,非法,含字母
```

## 7. 错误处理与回退

### 7.1 Faker 不可用时的回退策略

当 Faker 库未安装时，使用内置生成器：

| 字段类型 | 回退生成方式 |
|---------|-------------|
| 身份证 | 内置身份证生成器（预置地区码 + 随机日期 + 顺序码 + 校验位计算） |
| 手机号 | 预置号段 + 随机8位数字 |
| 用户名 | `testuser` + 随机数字 |
| 姓名 | 预置百家姓 + 随机2字名 |
| 邮箱 | `test` + 随机数字 + `@example.com` |
| 地址 | 预置省市 + 随机街道 + 随机门牌号 |
| 公司名 | 预置城市 + 预置字号 + 预置行业 + "有限公司" |
| 密码 | 自定义密码生成器 |
| URL | `https://example.com/` + 随机路径 |
| IP | 随机4段 0-255 |
| 银行卡号 | 随机16位数字 |
| 车牌号 | 省份简称 + 随机字母 + 随机5位 |

### 7.2 自然语言解析失败处理

当无法解析用户描述时：
1. 尝试提取数字和最接近的字段类型
2. 如果仍无法识别，返回解析错误信息和支持的字段类型列表
3. 建议用户使用更明确的描述方式
