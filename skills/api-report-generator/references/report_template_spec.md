# HTML 报告模板规范

## 设计风格

- **主色调**：科技蓝 `#1a73e8`
- **成功色**：`#34a853`
- **失败色**：`#ea4335`
- **跳过色**：`#9aa0a6`
- **警告色**：`#fbbc04`
- **背景色**：`#f8f9fa`
- **卡片背景**：`#ffffff`
- **文字主色**：`#202124`
- **文字辅色**：`#5f6368`
- **圆角**：`8px`
- **阴影**：`0 1px 3px rgba(0,0,0,0.12), 0 1px 2px rgba(0,0,0,0.08)`

## 页面分区详细规范

### 1. 报告头部 (Header)

```html
<header>
  <h1>{report_title}</h1>
  <div class="meta">
    <span>环境: {env}</span>
    <span>执行时间: {exec_time}</span>
    <span>生成时间: {create_time}</span>
    <a class="allure-btn" href="{allure_url}" target="_blank">打开Allure报告</a>
  </div>
</header>
```

- Allure按钮固定右上角，蓝色主色调，`target="_blank"` 新窗口打开
- 无Allure时按钮显示为灰色disabled状态

### 2. 总览大盘 (Overview)

四宫格卡片，每个卡片包含：
- 大号数字
- 小号标签文字
- 状态图标
- 通过率卡片带环形进度条

### 3. 趋势图表 (Charts)

- **通过率趋势**：折线图，X轴=执行序号/时间，Y轴=通过率(%)，数据点悬浮显示详情
- **模块分布**：环形饼图，按模块统计用例数占比
- **优先级柱状图**：按P0/P1/P2/P3分组，每组显示通过/失败堆叠柱

图表使用 Chart.js，配置参考 [chart_config.md](chart_config.md)

### 4. 模块统计表格 (Module Table)

| 列 | 说明 |
|----|------|
| 模块名称 | 如 auth、order、cart |
| 用例总数 | 该模块下用例数 |
| 通过数 | 绿色 |
| 失败数 | 红色，可点击展开失败详情 |
| 通过率 | 百分比 + 进度条 |
| 平均耗时 | 毫秒 |

### 5. 用例明细 (Case Detail)

- 表格列：用例名、接口路径、状态(图标)、响应时间、优先级标签、模块标签
- 顶部筛选栏：按状态(全部/通过/失败/跳过)、按模块、按优先级
- 分页：每页20条
- 行悬浮高亮，失败行红色背景

### 6. 故障详情 (Failure Section)

- 失败用例按模块分组
- 每个失败用例展示：
  - 用例名称（P0标红优先展示）
  - 错误信息（代码块格式）
  - AI诊断根因（来自 api-failure-diagnoser）
  - 修复建议
- 可折叠/展开

### 7. 数据清理记录 (Clean Record)

- 表格展示清理模块、清理前/后数量、清理条数、状态
- 底部汇总清理总条数

### 8. 风险分级 (Risk)

三级标识：
- 🔴 **高风险**：通过率 < 70%，P0用例失败
- 🟡 **中风险**：通过率 70%-90%，P1用例失败
- 🟢 **低风险**：通过率 > 90%，仅P2/P3失败

### 9. 优化建议 (Suggestion)

按优先级排序的卡片列表：
- 建议类型：脚本重构 / 断言调整 / 场景补充 / 环境优化
- 预期收益：提升通过率 X%
- 实施成本：低/中/高

### 10. 底部 (Footer)

```
接口自动化测试报告 | 生成工具: api-report-generator v1.0 | 报告时间: {time}
```

## 交互效果

1. **悬浮效果**：卡片、表格行 hover 时轻微上浮 + 阴影加深
2. **折叠展开**：故障详情、清理记录支持折叠/展开
3. **筛选**：用例明细顶部筛选栏，实时过滤
4. **分页**：用例列表每页20条，底部分页控件
5. **图表交互**：Chart.js 内置 tooltip、legend 点击切换

## 响应式断点

- `>= 1200px`：三栏图表并排
- `768px - 1199px`：图表两栏
- `< 768px`：图表单栏，表格横向滚动
