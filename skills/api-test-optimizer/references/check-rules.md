# 检查规则参考文档

本文档定义了 api-test-optimizer 执行的4类校验的详细规则，包含每个检查项的判定标准、严重级别和自动修复策略。

## 一、语法校验

### 1.1 Python 语法错误

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 语法错误 | `compile()` 抛出 SyntaxError | 🔴 致命 | 根据错误信息修复（补全冒号、修正括号、修复缩进） |
| 缩进错误 | AST 解析失败（Tab/空格混用、缩进层级不一致） | 🔴 致命 | 统一为4空格缩进，修正层级 |
| 编码错误 | 文件包含非 UTF-8 字符 | 🔴 致命 | 转换为 UTF-8 编码 |
| 未闭合字符串 | 三引号/引号未配对 | 🔴 致命 | 补全闭合引号 |

**检测方法：**

```python
import ast
import py_compile

def check_syntax(file_path):
    """语法检查"""
    issues = []
    # 1. 编译检查
    try:
        py_compile.compile(file_path, doraise=True)
    except py_compile.PyCompileError as e:
        issues.append(("🔴", "syntax", str(e), "auto_fix_syntax"))
    
    # 2. AST 解析检查（更细粒度）
    try:
        with open(file_path, "r", encoding="utf-8") as f:
            source = f.read()
        ast.parse(source)
    except SyntaxError as e:
        issues.append(("🔴", "syntax", f"L{e.lineno}: {e.msg}", "auto_fix_syntax"))
    
    return issues
```

### 1.2 导入缺失

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 模块未找到 | import 的模块在项目结构中不存在 | 🟡 警告 | 检查是否拼写错误，修正为正确模块名 |
| 循环导入 | A import B 且 B import A | 🟡 警告 | 重构导入关系，提取公共部分 |
| 未使用的导入 | import 了但未被引用 | 🔵 建议 | 删除未使用的 import 语句 |
| 星号导入 | `from module import *` | 🟡 警告 | 替换为显式导入 |

**检测方法：**

```python
import ast

def check_imports(file_path, project_structure):
    """导入检查"""
    issues = []
    with open(file_path, "r", encoding="utf-8") as f:
        tree = ast.parse(f.read())
    
    used_names = set()
    import_statements = []
    
    for node in ast.walk(tree):
        if isinstance(node, ast.Import):
            for alias in node.names:
                import_statements.append((alias.name, node.lineno))
        elif isinstance(node, ast.ImportFrom):
            for alias in node.names:
                import_statements.append((f"{node.module}.{alias.name}", node.lineno))
        elif isinstance(node, ast.Name):
            used_names.add(node.id)
    
    # 检查未使用的导入
    for imp, lineno in import_statements:
        module_name = imp.split(".")[-1]
        if module_name not in used_names:
            issues.append(("🔵", "import", f"L{lineno}: 导入 '{imp}' 未使用", "remove_import"))
    
    return issues
```

### 1.3 变量未定义

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|

### 1.4 函数签名不匹配

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |

| 关键字参数不存在 | 调用时传入的关键字参数在定义中不存在 | 🔴 致命 | 修正为正确参数名 |

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 缺少 requirements.txt | 项目中无 requirements.txt | 🟡 警告 | 自动生成 requirements.txt |

| 版本不兼容 | 包版本与代码使用方式不兼容 | 🔵 建议 | 更新 requirements.txt 版本号 |

## 二、规范校验

### 2.1 命名规则

| 检查项 | 规范要求 | 违规示例 | 正确示例 | 严重级别 |
|--------|---------|---------|---------|---------|
| 文件命名 | 小写字母+下划线 | `UserAPI.py` | `user_api.py` | 🟡 |
| 目录命名 | 小写字母+下划线 | `TestCases/` | `testcases/` | 🟡 |
| 类名 | 大驼峰命名法 | `user_api` / `userApi` | `UserAPI` | 🟡 |
| 方法名 | 小写字母+下划线 | `Login` / `get_User` | `login` / `get_user` | 🟡 |
| 测试方法前缀 | 必须以 `test_` 开头 | `check_login` | `test_login` | 🔴 |
| 变量名 | 小写字母+下划线 | `userName` / `UserName` | `user_name` | 🟡 |
| 常量名 | 全大写+下划线 | `base_url` / `BaseUrl` | `BASE_URL` | 🟡 |
| 测试类名 | `Test` + 大驼峰 | `test_user_login` / `testUserLogin` | `TestUserLogin` | 🔴 |
| 场景分类类名 | `Test` + 接口名 + 场景 | `TestLogin` | `TestUserLoginNormal` | 🟡 |

### 2.2 注释完整性

| 检查项 | 规范要求 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 模块 docstring | 每个 .py 文件必须有模块级 docstring | 🟡 | 自动生成：包含模块说明、接口路径、请求方法 |
| 类 docstring | 每个类必须有 docstring | 🟡 | 自动生成：包含类说明（接口名+场景分类） |
| 方法 docstring | 每个公开方法必须有 docstring | 🔵 | 自动生成：包含方法说明、参数说明 |

| 行内注释 | 复杂逻辑需有行内注释 | 🔵 | 对关键操作添加注释 |

```python
# 方法 docstring
"""
{api_name}
- 接口路径：{api_path}

    {param}: {param_description}

```

## 三、健壮性校验

### 3.1 超时配置

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 无超时配置 | 请求方法未设置 timeout 参数 | 🟡 | 注入 `timeout=TIMEOUT` |
| 超时值过小 | timeout < 5 秒 | 🟡 | 替换为 `TIMEOUT` 常量（默认30s） |
| 超时值过大 | timeout > 120 秒 | 🔵 | 建议调整为合理值 |
| 统一超时 | 是否使用全局常量而非硬编码 | 🟡 | 替换为常量引用 |

### 3.2 重试机制

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |

### 3.3 异常捕获

## 四、逻辑校验

### 4.1 三层断言

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 无断言 | 测试方法中无任何 assert 语句 | 🔴 | 注入三层断言 |
| 仅状态码断言 | 只有 status_code 断言 | 🔴 | 补充业务码+业务数据断言 |
| 仅状态码+业务码 | 缺少业务数据断言 | 🟡 | 补充关键字段断言 |
| 断言类型错误 | 使用 `assert response` 而非具体字段 | 🟡 | 替换为字段级断言 |
| 断言位置错误 | 断言在 try/except 外，异常时跳过 | 🟡 | 调整断言位置确保执行 |

**三层断言标准：**

```python
# ✅ 完整三层断言
AssertUtil.assert_all(
    response,
    expected_status=200,
    expected_code=0,
    field_checks=[
        {"field": "data.token", "check": "not_empty"},
        {"field": "data.user_id", "check": "type", "expect": int},
    ]
)

# ❌ 不合格断言
assert response.status_code == 200  # 仅状态码
assert True  # 无效断言
assert response  # 断言类型错误
```

### 4.2 断言有效性

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 恒真断言 | `assert True` / `assert 1==1` / `assert "ok"` | 🔴 | 替换为有效断言 |
| 永假断言 | `assert False` / `assert 0` | 🔴 | 修正为正确的条件断言 |
| 重复断言 | 同一字段多次相同断言 | 🔵 | 合并为一次断言 |
| 模糊断言 | 断言条件不明确（如 `assert data` 而非 `assert data["id"] == 1`） | 🟡 | 细化为具体字段断言 |

### 4.3 业务码断言

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 缺少业务码断言 | 未断言 `code` 字段 | 🟡 | 补充 AssertUtil.assert_business_code() |
| 业务码硬编码 | 业务码数字直接写在断言中 | 🔵 | 提取为常量或从数据中读取 |
| 错误码未覆盖 | 只断言成功码(0)，未覆盖错误码 | 🟡 | 异常用例补充对应错误码断言 |

### 4.4 业务数据断言

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 无数据断言 | 未验证响应体中的任何字段值 | 🟡 | 补充关键字段断言 |
| 非空断言不足 | 返回的关键字段未做 not_empty 检查 | 🟡 | 补充非空断言 |
| 类型断言不足 | 关键字段未做类型检查 | 🔵 | 补充类型断言 |
| 错误响应无断言 | 错误场景只断言 status_code | 🟡 | 补充错误信息断言 |

### 4.5 接口依赖

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
|--------|---------|---------|-------------|
| 前置接口失败 | 依赖的前置接口失败未处理 | 🟡 | 添加前置条件检查或 fixture 依赖 |
| 依赖数据未传递 | 上一个接口返回的数据未在下一个接口中使用 | 🟡 | 注入数据传递逻辑 |
| 依赖顺序错误 | 接口调用顺序不符合业务逻辑 | 🟡 | 调整用例执行顺序或添加依赖声明 |
| 循环依赖 | A 依赖 B，B 又依赖 A | 🔴 | 重构依赖关系 |

**依赖传递检测模式：**

```python
# 检测：接口A的返回值是否被接口B使用
# 模式1：response.json()["data"]["id"] → 传入下一个接口
# 模式2：fixture 返回值 → 作为参数传入
# 模式3：类变量 self.xxx → 在不同方法间传递
```

### 4.6 业务规则

| 检查项 | 判定标准 | 严重级别 | 自动修复策略 |
